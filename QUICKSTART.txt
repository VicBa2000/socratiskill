================================================================================
  SOCRATISKILL - Quick Start + How to Use
================================================================================

  A Claude Code plugin that turns the CLI into a socratic mentor: asks
  before answering, escalates hints on struggle, reviews past mistakes,
  flags antipatterns, and keeps a learning journal.


================================================================================
  1. QUICK START (fresh install, 3 minutes)
================================================================================

  Prerequisites:
    - bun        https://bun.com/docs/installation
    - node       (any recent LTS; already bundled with most setups)
    - Claude Code with plugin + hook support


  ------------------------------------------------------------------
  1.1 WHAT YOU CONFIGURE WHEN (read this first)
  ------------------------------------------------------------------

  Socratiskill installs at the USER level, not per project. Everything
  in 1.2 below is done EXACTLY ONCE per machine.

    [ONE-TIME]      Steps 1-3 in section 1.2.
                    Hooks come from the plugin manifest
                    (hooks/hooks.json), so they apply globally to
                    every Claude Code session in every project.
                    State lives in ~/.claude/socratic/.

    [PER PROJECT]   Nothing. The mentor follows you across every
                    project / working directory automatically. The
                    plugin manifest hooks fire even in projects that
                    have their own .claude/settings.json (which would
                    otherwise override user-level hook entries).

    [PER SESSION]   Nothing. Each new Claude Code console you launch
                    picks up the hooks and the plugin automatically.
                    You don't run `/plugin install` or anything else
                    on session start.

                    Per-session toggles you may want:
                      /socratiskill:socratic off    -> soft mute (keeps
                                                       hook running but
                                                       injects only a
                                                       30-token silencer)
                      /socratiskill:socratic pause  -> TRUE bypass (zero
                                                       tokens; renames
                                                       profile aside)
                      /socratiskill:socratic on     -> undo `off`
                      /socratiskill:socratic resume -> undo `pause`
                    All four are persisted across sessions.


  ------------------------------------------------------------------
  1.2 ONE-TIME SETUP (run these once per machine)
  ------------------------------------------------------------------

  Step 1. Clone the plugin repo anywhere you like, and REMEMBER the
  absolute path — you will pass it verbatim in Step 2.
    git clone https://github.com/VicBa2000/socratiskill ~/socratiskill

    The example above drops the repo at:
      Windows (Git Bash):  C:/Users/<you>/socratiskill
      macOS:               /Users/<you>/socratiskill
      Linux:               /home/<you>/socratiskill
    Clone wherever you prefer; just know the absolute path.

  Step 2. Tell Claude Code about the plugin (inside a Claude Code
  session). IMPORTANT: the `/plugin` CLI does NOT reliably expand
  `~` on Windows — always pass the ABSOLUTE path.

    /plugin marketplace add <absolute-path-to-repo>
    /plugin install socratiskill@socratiskill
    /reload-plugins

    Concrete examples (forward slashes work on Windows):
      /plugin marketplace add C:/Users/<you>/socratiskill
      /plugin marketplace add /Users/<you>/socratiskill
      /plugin marketplace add /home/<you>/socratiskill

    After `marketplace add`, verify the registration actually
    persisted before moving on:

      cat ~/.claude/plugins/known_marketplaces.json

    You should see a "socratiskill" entry with the exact path you
    passed. If it is NOT there, the add silently failed — Claude
    Code can print "Successfully added marketplace: socratiskill"
    even when the directory does not contain
    `.claude-plugin/marketplace.json` (for example because the `~`
    was taken literally, or you pointed at an empty/partial dir).
    Retry with a correct absolute path before Step 3.

  Step 3. Calibrate your pedagogical level (one question):
    /socratiskill:socratic calibrate
    -> pick 1..5 (Novice / Basic / Intermediate / Advanced / Expert).
    -> this also creates ~/.claude/socratic/profile.json the first time.

  You are done. From now on, every Claude Code console you open in
  any project responds as a socratic mentor adapted to your level.
  Keep coding normally.

  Optional: if for some reason you do NOT want to install via the
  plugin system (e.g. you want hook entries written directly into
  ~/.claude/settings.json), use the legacy installer instead of
  Step 2:
    bash ~/socratiskill/scripts/install.sh
  Do NOT mix both — running install.sh AND installing as a plugin
  would register the hooks twice, doubling the token cost per turn.


================================================================================
  2. HOW TO USE (daily cheatsheet)
================================================================================

  Everything is dispatched through /socratiskill:socratic <subcommand>.

  ------------------------------------------------------------------
  2.1 CHECK YOUR CURRENT STATE
  ------------------------------------------------------------------
    /socratiskill:socratic              -> status (same as below)
    /socratiskill:socratic status       -> enabled, level, mode, streak

    Reads: ~/.claude/socratic/profile.json

  ------------------------------------------------------------------
  2.2 TURN THE MENTOR ON / OFF / PAUSE
  ------------------------------------------------------------------
    /socratiskill:socratic off          -> SOFT mute (hook still runs;
                                           injects ~30-token DISABLED
                                           silencer telling the model
                                           to act as default Claude Code)
    /socratiskill:socratic on           -> undo `off`

    /socratiskill:socratic pause        -> TRUE bypass (renames
                                           profile.json aside; hook
                                           short-circuits at zero token
                                           cost, ~5ms)
    /socratiskill:socratic resume       -> undo `pause`

    When to use which:

      DEFAULT      full pedagogy, ~200-400 tokens/turn of context
      `off`        instant toggle, ~30 tokens/turn, state preserved
      `pause`      zero tokens/turn, state preserved in .paused file
      /plugin disable socratiskill       no hooks at all (Claude Code
                                         feature; reverse with /enable)
      /plugin uninstall socratiskill     full removal (also run
                                         scripts/uninstall.sh first if
                                         you used the legacy install)

    `off` and `pause` are persisted across sessions. The shell-script
    fallbacks bash scripts/pause.sh and bash scripts/resume.sh do the
    exact same thing — useful if the dispatcher is broken or you want
    to script it from a CI / wrapper.

  ------------------------------------------------------------------
  2.3 CHANGE THE LEVEL / MODE
  ------------------------------------------------------------------
    /socratiskill:socratic level 3      -> set user level (1..5)
    /socratiskill:socratic mode learn       -> prioritize teaching
    /socratiskill:socratic mode productive  -> prioritize speed

    Levels 1..5 in short:
      1 Novice        Live teacher        — explains everything
      2 Basic         Teacher with context — explains the why
      3 Intermediate  Pair programmer      — asks before writing
      4 Advanced      Code reviewer        — challenges decisions
      5 Expert        Silent colleague     — only flags real issues

  ------------------------------------------------------------------
  2.4 ASK FOR MORE / LESS GUIDANCE DURING A CONVERSATION
  ------------------------------------------------------------------
    /socratiskill:socratic hint         -> raise scaffolding (+1)
    /socratiskill:socratic faster       -> same as hint (alias, less filler)
    /socratiskill:socratic slower       -> drop scaffolding (-2, more socratic)
    /socratiskill:socratic challenge    -> next turn: anti-adulation mode

    These adjust THIS session's hint level. The mentor's automatic
    escalation (based on correct/wrong answers) continues in parallel.

  ------------------------------------------------------------------
  2.5 TEACH SOMETHING TO CLAUDE (Feynman mode)
  ------------------------------------------------------------------
    /socratiskill:socratic teach <topic>
    ... (you explain; Claude probes with questions, doesn't teach) ...
    /socratiskill:socratic endteach

    Example:
      /socratiskill:socratic teach closures
      > "A closure is a function that remembers variables from the
      > outer scope even after the outer function returned..."
      Claude: "Walk me through an example where two closures share
      the same outer variable but at different times?"
      ... after 3-4 turns ...
      /socratiskill:socratic endteach
      -> "teach ended: closures (Nmin, K gaps detected)"

    Gaps go to the session file and later to the daily journal.

  ------------------------------------------------------------------
  2.6 REVIEW PAST MISTAKES (spaced repetition, Leitner)
  ------------------------------------------------------------------
    /socratiskill:socratic review

    If you have a card due (an old topic you failed), Claude poses one
    verifiable question about it. Your answer updates the Leitner box
    (1 -> 3 -> 7 -> 14 -> 30 days). Two consecutive right answers
    advance the card; one wrong answer resets it.

    If no cards are due:
      "no review cards due"

  ------------------------------------------------------------------
  2.7 SEE WHAT YOU ACTUALLY LEARNED (journal)
  ------------------------------------------------------------------
    /socratiskill:socratic journal              -> daily (today)
    /socratiskill:socratic journal week         -> ISO week (Mon-Sun)
    /socratiskill:socratic journal month        -> calendar month

    Generates a markdown rollup from your session files, categorizing
    topics into Learned / Struggled / Practiced, plus any Feynman
    teach summaries and a Leitner snapshot. Saved under
    ~/.claude/socratic/journal/.

  ------------------------------------------------------------------
  2.8 ACCEPT A CALIBRATION SUGGESTION
  ------------------------------------------------------------------
    After 3+ correct or 3+ wrong answers in a 5-turn window, the
    mentor suggests adjusting your level. When you see "calibration:
    suggest level X -> Y" in context, run:

    /socratiskill:socratic accept

    This applies the change and resets the session's hint counters.


================================================================================
  3. TYPICAL WORKFLOWS
================================================================================

  ------------------------------------------------------------------
  3.1 DAILY CODING (level 3 or 4, mode productive)
  ------------------------------------------------------------------
    Start: /socratiskill:socratic mode productive
    Work normally. Claude asks before writing, flags issues, answers
    without hand-holding.
    End of day: /socratiskill:socratic journal
    -> see what you practiced and where you struggled.

  ------------------------------------------------------------------
  3.2 LEARNING A NEW TOPIC (level 2 or 3, mode learn)
  ------------------------------------------------------------------
    Start: /socratiskill:socratic mode learn
    Ask the topic. Claude explains with verification questions.
    If it goes too fast: /socratiskill:socratic slower
    If too slow:        /socratiskill:socratic faster
    Once you think you get it:
      /socratiskill:socratic teach <topic>
      (explain it in your own words; Claude probes for gaps)
      /socratiskill:socratic endteach

  ------------------------------------------------------------------
  3.3 WEEKLY REFRESHER
  ------------------------------------------------------------------
    Every few days:
      /socratiskill:socratic review
      (answer the card; Claude schedules the next review)
    Once a week:
      /socratiskill:socratic journal week
      (see which topics resurfaced as "struggled")


================================================================================
  4. WHERE STATE LIVES
================================================================================

  ~/.claude/socratic/profile.json          your level, mode, enabled flag
  ~/.claude/socratic/error-map.json        Leitner boxes per topic
  ~/.claude/socratic/antipatterns.json     active antipatterns + counts
  ~/.claude/socratic/sessions/<date>.json  one file per UTC day, telemetry
  ~/.claude/socratic/journal/<file>.md     generated rollups


================================================================================
  5. TROUBLESHOOTING
================================================================================

  Problem: /socratiskill:socratic status shows "enabled: false".
    Fix:   /socratiskill:socratic on

  Problem: Subcommands behave weird after editing SKILL.md.
    Fix:   /reload-plugins (Claude Code caches the SKILL.md body).
           Scripts (*.ts, *.sh) are picked up on every call — no reload
           needed for those.

  Problem: "Welcome to Socratiskill" never shows up.
    Fix:   You already calibrated. To redo:
             /socratiskill:socratic calibrate force

  Problem: "no review cards due" but you know you have struggled topics.
    Fix:   Open ~/.claude/socratic/error-map.json. A card is due when
           next_review_at <= now AND resolved == false. If everything
           is upcoming, you just have to wait.

  Problem: HINT_META block shows up as visible text in your client.
    Cause: Your markdown renderer doesn't strip HTML comments.
    Fix:   The block is harmless telemetry. Open an issue; we might
           add an opt-in alternative format.

  Problem: You want to start completely fresh.
    Fix:   bash ~/socratiskill/scripts/uninstall.sh --purge
           Then /plugin install socratiskill@socratiskill again
           and /socratiskill:socratic calibrate.

  Problem: Hooks fire twice per turn (you see two SOCRATIC CONTEXT
           blocks, or counters are doubling).
    Cause: You ran both `bash scripts/install.sh` AND installed via
           `/plugin install`, so the hooks are registered twice
           (once in ~/.claude/settings.json, once via the plugin
           manifest).
    Fix:   bash ~/socratiskill/scripts/uninstall.sh --keep-state
           This removes the settings.json entries; the plugin
           manifest hooks remain and you have a single firing.

  Problem: Plugin appears to be doing nothing in a specific project.
    Cause: That project has its own .claude/settings.json. Before
           v0.2 the user-level hooks did not propagate; now they do
           because plugin manifest hooks are global. If you are still
           on v0.1 or used the legacy install path, run:
             cd <that-project>
             CLAUDE_SETTINGS="$(pwd)/.claude/settings.json" \
               bash ~/socratiskill/scripts/install-hooks.sh

  Problem: I want zero token cost without uninstalling.
    Fix:   /socratiskill:socratic pause
           Resume any time with /socratiskill:socratic resume.

  Problem: /plugin install says "Plugin 'socratiskill' not found in
           any marketplace" right after a "Successfully added
           marketplace" message.
    Cause: The `marketplace add` actually failed. Claude Code prints
           a success line even when the path you gave does not
           contain .claude-plugin/marketplace.json (for example, if
           `~` was not expanded by the CLI, you typed a path that
           only partially exists, or the directory is empty).
    Fix:   cat ~/.claude/plugins/known_marketplaces.json
           If "socratiskill" is NOT listed, re-run with an absolute
           path and forward slashes (no tilde):
             /plugin marketplace add C:/Users/<you>/socratiskill
             /plugin install socratiskill@socratiskill
           Then verify again. `~/.claude/plugins/known_marketplaces.json`
           is the source of truth.


================================================================================
  6. DISABLE / PAUSE / UNINSTALL
================================================================================

  Four levels of "stop the plugin", from softest to heaviest:

  6.1 SOFT MUTE (state preserved, ~30 tokens/turn for silencer):
    /socratiskill:socratic off
    /socratiskill:socratic on        (to re-enable)

  6.2 TRUE BYPASS (state preserved, zero token cost):
    /socratiskill:socratic pause
    /socratiskill:socratic resume    (to come back)

    Equivalent shell scripts (useful from CI / scripts):
      bash ~/socratiskill/scripts/pause.sh
      bash ~/socratiskill/scripts/resume.sh

  6.3 PLUGIN-LEVEL DISABLE (Claude Code feature):
    /plugin disable socratiskill
    /plugin enable socratiskill      (to re-enable)

  6.4 FULL UNINSTALL:
    bash ~/socratiskill/scripts/uninstall.sh
    (asks interactively whether to keep your journal/profile; use
    --keep-state or --purge to skip the prompt; --dry-run to preview)

    Then, inside Claude Code:
    /plugin uninstall socratiskill

  Notes:
    - With the v0.2 plugin manifest install, the hooks are registered
      via hooks/hooks.json and NOT in ~/.claude/settings.json. The
      uninstall.sh hook-removal step only matters if you used the
      legacy install path.
    - uninstall.sh has hardened guards: it refuses to rm -rf any
      SOCRATIC_STATE_DIR that is not absolute, not under $HOME, lacks
      the .claude/socratic segment, or contains a `..` traversal.
